# Interview-Driven Development Template

Шаблон для разработки проектов через структурированные AI-интервью.

## Идея

придумал -> прошёл интервью с AI-продактом -> прошёл интервью с AI-архитектором -> получил specs -> запустил агентский кодинг -> повторил.

## Quick Start

В шаблоне есть **3 TODO**, которые нужно заполнить перед стартом разработки:

### 1. Продуктовое интервью -> `specs/PRD.md`

Откройте новый диалог с AI и передайте ему содержимое `specs/roles/PRD_ROLE.md` как системный промпт (или вставьте первым сообщением). В конце промпта в секции "Моя идея" опишите свой продукт. AI проведёт глубокое структурированное интервью: по одному вопросу за раз, от vision до edge cases. В финале сгенерирует итоговый PRD, поместите результат в `specs/PRD.md` вместо `TODO`.

### 2. Техническое интервью -> `specs/TECHSPEC.md`

Откройте новый диалог и передайте содержимое `specs/roles/TECHSPEC_ROLE.md`. Первым сообщением передайте готовый PRD из шага 1. AI в роли CTO/Principal Architect проведёт техническое интервью: от стека и ограничений до архитектуры, data model, API и рисков. В финале сгенерирует итоговый TECHSPEC, поместите результат в `specs/TECHSPEC.md` вместо `TODO`.

### 3. Правила кодинга -> `specs/AGENTS.md`

Если у вас есть свои правила и конвенции кодинга — добавьте их в `specs/AGENTS.md` вместо `TODO`. Это могут быть: стек, стиль кода, ограничения по размеру файлов, правила тестирования, предпочтения по библиотекам. В файле уже есть примеры для C#/.NET и React.

### 4. Milestones -> `specs/MILESTONES.md`

Начиная с этого шага работайте через локального AI-агента с доступом к файловой системе (Claude Code, Cursor, etc.) — он сам найдёт нужные файлы и запишет результат. Передайте агенту содержимое `specs/roles/MILESTONE_ROLE.md` как промпт. Агент прочитает PRD и TECHSPEC, разобьёт проект на крупные этапы и запишет результат в `specs/MILESTONES.md`.

### 5. Backlog -> `specs/backlog/ACTIVE.md`

Передайте локальному агенту содержимое `specs/roles/BACKLOG_ROLE.md`. Агент сам подтянет PRD, TECHSPEC и MILESTONES, декомпозирует первый milestone на конкретные задачи и наполнит `specs/backlog/ACTIVE.md`.

### 6. Разработка

Запустите coding-агента одним из способов:

- **Вручную** — откройте диалог с `specs/roles/CODING_ROLE.md` и работайте по задачам из backlog
- **Автономно** — запустите `./ralf.sh <N>`, где `N` — количество задач, которые агент должен выполнить. Скрипт сам будет брать задачи из backlog и закрывать их в цикле. Если скрипт завис, значит необходимо включить vpn.

## Структура specs/

### Канонические документы (source of truth)

| Файл | Назначение |
|---|---|
| `PRD.md` | **Что** и **зачем** — vision, user flows, scope, success criteria |
| `TECHSPEC.md` | **Как** — архитектура, data model, API, NFR, интеграции |
| `MILESTONES.md` | Roadmap — крупные этапы реализации |
| `ADR/` | Architecture Decision Records — зафиксированные архитектурные решения с trade-offs |
| `AGENTS.md` | Универсальные правила кодинга для AI-агентов |

### Операционный слой

| Файл | Назначение |
|---|---|
| `STATE.md` | Текущее состояние проекта |
| `backlog/` | Задачи: `INBOX.md` (входящие), `ACTIVE.md` (в работе), `archive/` (выполненные) |
| `tasks/` | Детализированные задачи |

### Основные роли (промпты для интервью и работы)

| Файл | Назначение |
|---|---|
| `roles/PRD_ROLE.md` | Промпт для продуктового интервью (Lead PM) |
| `roles/TECHSPEC_ROLE.md` | Промпт для технического интервью (CTO/Architect) |
| `roles/CODING_ROLE.md` | Промпт для coding-агента |
| `roles/MILESTONE_ROLE.md` | Формирование roadmap |
| `roles/BACKLOG_ROLE.md` | Декомпозиция на задачи |
| `roles/SPEC_ROLE.md` | Синхронизация specs после реализации (ужатие шума, обновление ссылок, etc.) |

### Дополнительные роли (brownfield и доработки)

| Файл | Назначение |
|---|---|
| `roles/INBOX_ROLE.md` | Приём и маршрутизация входящих идей (tiny/major) |
| `roles/CHANGE_INTAKE_ROLE.md` | Оформление change pack для крупного изменения |
| `roles/PRD_DELTA_ROLE.md` | Генерация продуктовой дельты к существующему PRD |
| `roles/TECHSPEC_DELTA_ROLE.md` | Генерация технической дельты к существующему TECHSPEC |
| `roles/CHANGE_MERGE_ROLE.md` | Контролируемый merge дельт в канонические документы |
| `roles/DEV_SYNC_ROLE.md` | Синхронизация specs с фактическим состоянием кода после ручной реализации |

### Change management (brownfield)

| Файл | Назначение |
|---|---|
| `changes/` | Change packs для крупных изменений в существующем продукте |
| `OPERATOR.md` | Гайд по ведению specs в greenfield и brownfield режимах |

### Автоматизация

| Файл | Назначение |
|---|---|
| `ralf.sh` | Скрипт автономного цикла разработки — запускает coding-агента в цикле по задачам из backlog |
