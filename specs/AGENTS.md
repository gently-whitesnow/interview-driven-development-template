# AGENTS — Universal Coding Conventions

Этот файл — портируемые правила кодинга. Project-specific нюансы и стек — в `CLAUDE.md`. Здесь только то, что верно для любого проекта.

## General

- Файл больше 300 строк — декомпозируй.
- При интеграции со сторонним сервисом сверяйся с актуальной документацией (websearch / official docs), не на память.
- Не изобретай велосипеды. Если решение существует в проекте или в стандартной библиотеке — используй его.
- Третье независимое повторение — повод для абстракции (rule of three). Дублирование двух кусков дешевле неверной абстракции.

### example: C# / .NET

- Старайся в нагруженных частях возвращать кортеж (статус ошибки, ожидаемый результат), а не выбрасывать исключения.
- В API не отдавать enum, вместо этого используй строковое значение.
- Для enum-like API полей фиксируй один wire-формат (например `snake_case`) и на backend/frontend используй только его.
- Для фронтового API вместо long возвращай string.
- Используй primary конструктор.
- Пиши `[Fact(DisplayName =` для тестов с кратким описанием теста на русском языке.
- Не используй в тестах `await Task.Delay(...)`.
- Если метод возвращает задачу TASK, то в его названии должно быть Async (SaveAsync, CreateAsync, UpdateAsync, DeleteAsync, etc.)
- DI: по умолчанию `AddSingleton`. `AddScoped`/`AddTransient` — только если есть mutable state, per-request данные или non-thread-safe зависимости.
- Не инжекти `IServiceProvider` — используй прямые зависимости через конструктор. Service Locator допустим только при динамическом resolve (например, по ключу из конфига).

### example: React / Frontend

- **Light-first**: dark поддерживается, но дефолт — светлая тема.
- Для статусов используй семантические цвета (success/warning/error/info), не хардкодь hex.
- Иконки — только `lucide-react`.
