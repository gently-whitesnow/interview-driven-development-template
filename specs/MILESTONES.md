# MILESTONES — Implementation Plan

Этот файл хранит baseline roadmap и крупные milestone-level increments.

## Maintenance Rule
- Не обязательно расписывать здесь каждый tactical patch, hotfix или небольшой brownfield slice.
- Для существенных brownfield changes сначала используй `specs/OPERATOR.md` и `specs/changes/**`; `MILESTONES.md` патчится только если изменение реально влияет на roadmap, sequencing или появляется новый milestone-level increment.
- Если новый milestone появляется в pipeline, его можно добавить кратко и адресно. Не регенерируй весь файл без необходимости.

Текущий baseline включает M1-M13. M2 и M3 могут вестись параллельно; остальные базовые milestones — последовательно, если в секции `Dependencies / Notes` не указано иное.
